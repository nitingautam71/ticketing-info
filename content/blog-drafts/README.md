# Blog drafts

Generated by `.github/workflows/daily-blog.yml` (Claude Code, headless, daily at 08:00 IST).
These JSON files are a git-reviewable staging area only - the live blog at `/blog` reads from
the `BlogPost` table in Postgres (see `prisma/schema.prisma`), not from this directory. A file
here has no effect on the site by itself.

## Flow

1. Daily workflow researches travel news and writes one file here: `YYYY-MM-DD-<slug>.json`.
2. It opens a PR (`blog-drafts/YYYY-MM-DD` -> `main`) containing that file, with the full post
   text in the PR description, and immediately auto-merges it - by explicit request, there is no
   human review gate here. The PR exists only as a git history / audit trail, not a checkpoint.
3. That merge triggers `.github/workflows/publish-blog-on-merge.yml`, which calls the site's
   `/api/admin/blog` endpoint to create the post live with `published: true`. That's the only
   step that touches production, and it now happens automatically, same-day, with no manual step.

## File shape

```json
{
  "date": "2026-07-12",
  "slug": "irctc-tatkal-booking-rules-change",
  "title": "SEO title, under ~70 characters",
  "excerpt": "Meta description / card teaser, under ~155 characters",
  "content": "Plain text body, paragraphs separated by a blank line, ending with a 'Sources' section.",
  "sources": [{ "title": "Publisher - article title", "url": "https://..." }]
}
```

Files are kept after publishing as a dated history of what ran each day; they're not deleted or
otherwise consumed after the merge workflow runs.
